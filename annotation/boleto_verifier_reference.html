Pacotes instalados NuGet
Swashbuckle.AspNetCore
Gera a interface do Swagger automaticamente a partir dos seus Controllers. Permite testar os endpoints direto no navegador sem precisar do Postman.
MailKit
Biblioteca para envio de e-mails via SMTP. Mais robusta que o SmtpClient nativo do .NET — suporta TLS, autenticação OAuth e envio assíncrono.
Conceitos do .NET utilizados .NET / C#
Injeção de dependência
Padrão onde o .NET cria e entrega os objetos que uma classe precisa. Registrado com AddScoped<T>() no Program.cs.
AddScoped
Define que uma nova instância do service é criada por requisição HTTP. Ideal para services que leem configuração ou fazem I/O.
IConfiguration
Interface que lê configurações do appsettings.json e dos User Secrets. Usada no EmailService para buscar e-mail e senha.
User Secrets
Armazena credenciais fora do código, no sistema operacional local. Nunca vai para o repositório Git. Acessado via _config["Chave"].
CORS
Política de segurança do navegador que bloqueia chamadas entre origens diferentes. Liberado na API com AddCors() e UseCors().
ApiController / Route
Atributos que transformam uma classe C# em um endpoint REST. [Route("api/[controller]")] define a URL automaticamente pelo nome da classe.
async / await
Permite que a API não trave enquanto espera operações lentas (envio de e-mail, I/O). Método retorna Task<IActionResult> em vez de IActionResult.
Comandos de terminal CLI
dotnet new webapi -n nome Cria um novo projeto de API REST com template padrão.
dotnet run Compila e inicia o servidor localmente.
dotnet run --launch-profile http Inicia usando o perfil HTTP definido no launchSettings.json.
dotnet add package Nome Instala um pacote NuGet no projeto (equivalente ao npm install).
dotnet user-secrets init Habilita o armazenamento de segredos locais no projeto.
dotnet user-secrets set "Chave" "Valor" Salva uma credencial segura fora do código-fonte.
dotnet dev-certs https --trust Instala e confia no certificado HTTPS local gerado pelo .NET.
Conceitos de JavaScript utilizados JS
fetch API
Função nativa do navegador para fazer requisições HTTP. Substitui o antigo XMLHttpRequest. Retorna uma Promise.
async / await
Permite escrever código assíncrono de forma linear, sem callbacks aninhados. O await pausa a execução até a Promise resolver.
try / catch / finally
Trata erros em código assíncrono. O bloco finally executa sempre — usado para reabilitar o botão após o envio.
JSON.stringify
Converte um objeto JavaScript em texto JSON para enviar no corpo da requisição HTTP.
response.json()
Converte a resposta da API (texto JSON) de volta para um objeto JavaScript.
